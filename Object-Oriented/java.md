# JAVA의 특징


```
자바(Java)는 객체 지향 프로그래밍 언어로, 1995년에 선 마이크로시스템즈
(현재 오라클)에 의해 개발되었습니다. 초기에는 가전 제품 내 소프트웨어 개발에 
사용하기 위해 고안되었으며, 이후 웹 및 엔터프라이즈 애플리케이션 개발에
매우 널리 사용되는 언어가 되었습니다.
```
<br>

## 1.객체 지향 언어
```
자바는 대표적인 객체지향 프로그래밍 언어입니다.
실제세계를 객체라는 단위로 나누고 객체들간의 상호작용을 의미합니다.
즉,프로그램을 여러개의 독립적인 단위인 객체들의 모임으로 파악하고 하는 것입니다.
```
<br>

## 2.운영체제에 독립적으로 동작
```
 "Write once, run anywhere"

운영체제의 종류에 관계없이 실행 가능합니다.
자바 가상 머신(JVM)을 통해서 가능한 일입니다.

자바 가상 머신 : 자바 프로그램은 먼저 자바 컴파일러를 사용하여 바이트 코드(bytecode)로 변환됩니다.
 이 바이트 코드는 특정 운영체제가 아닌 JVM에서 실행됩니다. 
 JVM은 운영체제와 자바 프로그램 사이의 중간 계층 역할을 수행하며,
운영체제에 종속되는 부분을 처리합니다. 
각 운영체제에 맞게 설계된 JVM이 있기 때문에 동일한 바이트 코드를 
다른 운영체제의 JVM에서 실행할 수 있습니다.

그러나, JVM은 운영체제에 종속적이다.썬에서는 여러 운영체제에 
설치할 수 있는 서로 다른 버전의 JVM을 제공하고 있다.
```
<br>


## 3.자동 메모리 관리

```
다른 프로그래밍 언어와는 달리, 자바는 가비지 컬렉터(GC)가 자동으로 메모리를 관리하여 
참조되고 있지 않은 메모리를 해제해줍니다.

이러한 특징으로 인해 프로그래머는 메모리 관리에 신경 쓸 필요 없이 
오직 프로그래밍에만 집중 할 수 있습니다.
```
<br>

## 4.멀티쓰레드 지원
```
하나의 프로그램 단위가 동일한 쓰레드를 동시에 수행할 수 있습니다.

자바의 멀티쓰레드는 시스템과 관계없이 구현이 가능하며, Java API를 통해 쉽게 구현할 수 있습니다.

자바의 멀티쓰레드 지원은 여러 스레드를 활용하여 동시에 작업을 수행하고
병렬성을 활용할 수 있도록 합니다. 이는 프로그램의 성능을 향상시키고
복잡한 작업을 더 효율적으로 처리할 수 있도록 도와줍니다. 
```

<br>

## 5.동적 로딩의 지원
```
동적로딩은 프로그램 실행 중에 클래스를 동적으로 로드하는 기능을 의미합니다. 
이는 런타임 시에 필요한 클래스를 동적으로 로드하여 사용할 수 있게 해주는 기능으로, 
자바의 유연성과 확장성을 높이는 데 도움을 줍니다.

자바의 동적로딩은 주로 클래스로더(ClassLoader)와 리플렉션(Reflection)을 이용하여 구현됩니다.

클래스로더(ClassLoader): 클래스로더는 자바 런타임 환경에서 클래스를
동적으로 로드하는 역할을 수행합니다. 

자바에서는 기본적으로 세 가지 클래스로더가 제공됩니다
1.부트스트랩 클래스로더  2.확장 클래스로더 3.애플리케이션 클래스로더

부트스트랩 클래스로더는 자바 런타임 자체를 로딩하는 역할을 합니다 
확장 클래스로더는 시스템 확장 영역의 클래스를 로딩합니다. 
애플리케이션 클래스로더는 애플리케이션 클래스패스에 있는 클래스를 로딩합니다. 
개발자는 커스텀 클래스로더를 작성하여 동적로딩을 구현할 수도 있습니다.

리플렉션(Reflection): 리플렉션은 프로그램이 런타임에 자기 자신의 구조를 분석하고 
수정할 수 있는 기능을 의미합니다. 

자바에서는 Class 클래스와 java.lang.reflect 패키지를 통해 리플렉션 기능을 제공합니다. 

리플렉션을 이용하면 동적으로 클래스를 로드하고, 클래스의 필드, 
메서드, 생성자 등에 접근할 수 있습니다. 
이를 통해 동적으로 생성된 객체를 다루거나 클래스의 인스턴스를 생성할 수 있습니다.

단, 동적로딩은 정적으로 컴파일된 클래스 파일에 비해 성능상의 오버헤드가
발생할 수 있으며, 적절한 예외 처리와 보안 검사가 필요합니다. 
또한, 잘못된 사용은 코드 가독성을 낮출 수 있습니다.
```
